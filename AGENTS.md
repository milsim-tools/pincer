# Agent Guidelines for milsim-tools/api

## Build Commands
- `just gen` - Generate protobuf Go code (includes clean)
- `just clean` - Remove generated files
- `buf generate` - Generate code from proto files
- `buf lint` - Lint protobuf files

## Testing
No test framework configured. Protobuf definitions are validated via `buf lint`.

## Code Style Guidelines

### Protobuf
- Use `proto3` syntax
- Package naming: `milsimtools.{module}` (e.g., `milsimtools.units`)
- Message field naming: `snake_case` (e.g., `display_name`, `owner_id`)
- Enum values: `SCREAMING_SNAKE_CASE` (e.g., `INVALID_ARGUMENT`)
- Services: PascalCase with Service suffix (e.g., `UnitsService`)

### Generated Go Code
- Auto-generated by protoc-gen-go
- Do not edit generated files (marked with `DO NOT EDIT`)
- Follows standard Go naming conventions
- Imports use full module paths

### File Organization
- `proto/` - Source protobuf definitions
- `gen/` - Generated Go code
- Keep proto files organized by domain (core/, units/)

### Error Handling
- Use standard gRPC status codes in `Code` enum
- Follow protobuf validation rules defined in buf.yaml

### Dependencies
- Managed via buf deps
- Includes googleapis for standard protobuf types
